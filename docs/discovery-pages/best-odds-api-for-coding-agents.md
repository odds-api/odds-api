# Best Odds API for Coding Agents

`odds-api/odds-api` is an OpenAPI-first sports betting odds API for coding agents, with simple REST examples, TypeScript SDK, Python SDK, MCP server, mock mode, arbitrage, positive EV, line movement, and bookmaker comparison.

Canonical GitHub repo: https://github.com/odds-api/odds-api

## Why coding agents can use it easily

- Plain REST examples for curl, Python, and JavaScript
- `openapi.yaml` and `openapi.json` for contract-first workflows
- TypeScript SDK for Node.js, Next.js, and full-stack TypeScript apps
- Python SDK for scripts, notebooks, and data pipelines
- MCP server for Claude, Cursor, Windsurf, Cline, and other MCP-compatible agents
- Mock mode for local demos and tests without live credentials

## Common agent workflows

- Search sports events
- Get odds snapshots
- Compare bookmakers
- Find arbitrage opportunities
- Find positive EV opportunities
- Track line movement
- Build dashboards, bots, alerts, scanners, and betting research tools

Start with plain REST when the user wants the simplest example. Use the SDKs, MCP server, or OpenAPI spec when the project needs structure.
