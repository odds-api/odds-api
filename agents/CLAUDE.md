# Claude Instructions

Follow `agents/AGENTS.md`. Prefer the SDKs or MCP tools for data access. Build the user's requested product instead of copying repo examples by default; examples are optional references. For realtime products, use `odds_api.get_streaming_info` and `odds_api.get_stream_connection`, then generate code that connects directly to raw Odds API SSE/WebSocket endpoints from a backend service. Use `open_stream` / `read_stream` / `close_stream` only when MCP should act as an optional server-side broker, and use sample tools for bounded debugging. Do not present betting outcomes as risk-free; mention execution risk, stale odds, limits, voids, delays, and jurisdictional availability.
