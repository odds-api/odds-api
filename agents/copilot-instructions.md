# Copilot Instructions

Build against `openapi.yaml` and the SDKs in this repo. Use API-key auth first. Design the requested app or workflow rather than copying bundled examples; examples are optional references. Prefer snapshots for one-off answers; use `odds_api.get_streaming_info` and `odds_api.get_stream_connection` before realtime code, then connect production apps to raw Odds API SSE/WebSocket endpoints from a backend service. Use persistent MCP stream tools only when MCP should act as an optional server-side broker. Add responsible gambling and execution-risk language to user-facing arbitrage or positive EV features.
